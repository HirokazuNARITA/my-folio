# ワークフロー例

## 例1: ページ追加（軽い複雑さ）

```
1. app/admin/works/new/page.tsx を作成
2. npm run build → 成功確認
3. npm run dev 起動、ブラウザで /admin/works/new を開く
4. 期待する文言・レイアウトが表示されるか確認
5. verify:pages に該当パスを追加し、npm run verify:pages で自動確認
6. 次の実装へ
```

## 例2: フォーム実装（中〜複雑）

```
1. バリデーションロジック（Zod schema）を実装
   → ユニットテストを書く（RED）→ 実装（GREEN）→ リファクタ
2. フォームコンポーネントを実装
   → build/lint 確認 → ブラウザでレンダリング確認
3. Server Action を実装
   → Action のテスト or 手動送信で確認
4. エラーハンドリング追加
   → エラーケースのテスト → 実装 → ブラウザで確認
5. 最終: 全フローをブラウザで一通り実行
```

## 例3: API 実装（複雑・TDD）

```
1. テストを書く: 期待するリクエスト→レスポンス
2. 実装: テストが通る最小コード
3. リファクタ
4. 統合: フロントから呼び出し、ブラウザ or E2E で確認
```
